Bloques personalizados

El editor expone una API mínima para registrar bloques nuevos. Cada bloque debe implementar:

  • Un schema tipado en TypeScript que extiende BlockBase.
  • Un emisor MJML en compileToMjml (o reusa renderText).
  • Una validación adicional en validators.ts si el bloque tiene reglas propias.

Los tipos válidos en el MVP están enumerados en ALLOWED_BLOCK_TYPES (TS) y JsonSchemaValidator::ALLOWED_BLOCK_TYPES (PHP). Para bloques de terceros, usa el filtro imgsig/signature/data_before_save para sanear y el hook imgsig/services/registered para registrar handlers.
